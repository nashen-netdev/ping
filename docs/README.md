## 📖 文档规范说明

### 为什么文档分为两类？

遵循业界最佳实践，文档分为：

1. **根目录核心文档**（3-4个）
   - 这些是最常访问的文档
   - 必须放在根目录方便查找
   - 例如：README、CHANGELOG、CONTRIBUTING、LICENSE

2. **docs/详细文档**（技术文档）
   - 详细的技术说明
   - 开发者和高级用户需要的文档
   - 例如：安装、开发、测试、架构

### 文档组织原则

- ✅ **根目录简洁**：只放3-4个最核心的MD文档
- ✅ **详细文档归档**：技术文档都在docs/
- ✅ **历史记录分离**：一次性文档放history/
- ✅ **易于查找**：重要文档在根目录，详细文档有导航

### 参考标准

本项目文档结构参考了以下著名Python项目：

- **Django**：根目录3个MD，详细文档在docs/
- **Flask**：根目录4个MD，详细文档在docs/
- **Requests**：根目录4个MD，详细文档在docs/
