## 视频 1：安装和基础使用（5 分钟）

### 场景设定
- 演示环境：macOS（Windows 用户看注释）
- 目标用户：完全不懂编程的人
- 时长：5 分钟

### 脚本

**[0:00-0:30] 开场**
```
大家好，今天教大家如何使用这个网络 Ping 测试工具。
这个工具可以帮你批量测试网络设备是否能 ping 通。

即使你完全不懂编程，跟着我的步骤，5 分钟就能学会。
```

**[0:30-1:00] 检查前提条件**
```
首先，确保你的电脑安装了 Python。

打开终端（Windows 用户打开命令提示符）
输入：python3 --version
如果看到版本号，说明已经安装了。

如果没有，访问 python.org 下载安装即可。
```

**[1:00-2:00] 下载和进入项目**
```
下载项目后，用终端进入项目目录。

我把项目放在了 Documents 文件夹：
cd ~/Documents/ping

检查是否进入正确：
pwd

看到项目路径就对了。
```

**[2:00-3:30] 一键安装**
```
接下来是最重要的一步：安装依赖。

复制这行命令，粘贴到终端，回车：
python3 -m venv .venv && source .venv/bin/activate && pip3 install -r requirements.txt

等待几秒钟，看到 "Successfully installed" 就说明安装成功了。

如果报错，可以查看文档中的故障排查部分。
```

**[3:30-4:30] 第一次运行**
```
现在我们来测试一个 IP 地址规划表。

首先，把你的 Excel 文件放到 pass 文件夹。
我这里已经有一个 IP地址规划表.xlsx 文件。

然后运行：
ping-ip-planning

看，出现了交互式界面。
```

**[4:30-5:00] 查看结果**
```
按提示输入信息：
1. 文件路径
2. 选择 Sheet
3. 是否过滤颜色

等待测试完成，就能看到结果了。

详细日志保存在 logs 文件夹，可以随时查看。

就是这么简单！
```

## 视频 2：进阶功能（5 分钟）

### 场景设定
- 前置：已看过视频 1
- 目标：学会使用高级功能
- 时长：5 分钟

### 脚本

**[0:00-0:30] 回顾和目标**
```
上次我们学会了基本使用。
今天教大家几个实用的高级功能：
1. 环境配置管理
2. 从服务器 ping
3. 延迟分析
```

**[0:30-2:00] 环境配置管理**
```
如果你经常测试多个项目，可以为每个项目创建配置。

使用这个命令：
ping-env-add bj08 pass/北京机房.xlsx --display-name "北京08机房"

创建完成后，下次直接运行：
ping-ip-planning

选择对应的环境即可，非常方便。

查看所有环境：
ping-ip-planning --list-profiles
```

**[2:00-3:30] 从服务器 ping**
```
这是一个很实用的功能。

如果你需要从某台服务器测试网络，
不用登录到那台服务器，工具会自动帮你连接。

运行：
ping-ip-planning

选择 server&security sheet
选择 "从服务器 ping"
输入服务器的 hostname 或 IP

工具会自动从 Excel 读取登录凭据，
然后从那台服务器 ping 其他设备。
```

**[3:30-4:30] 查看延迟分析**
```
测试完成后，打开日志文件。

你会看到详细的延迟统计：
- 最小延迟
- 平均延迟  
- 最大延迟
- 延迟抖动

还会自动标记出延迟较高的 IP。

这些信息对网络优化很有帮助。
```

**[4:30-5:00] 总结**
```
今天我们学习了三个高级功能。

记住这些技巧：
1. 创建环境配置，提高效率
2. 从服务器 ping，无需手动登录
3. 关注延迟分析，优化网络

更多功能请查看完整文档。
```

## 视频 3：故障排查（3 分钟）

### 场景设定
- 目标：解决常见问题
- 时长：3 分钟

### 脚本

**[0:00-0:20] 开场**
```
使用过程中遇到问题怎么办？
今天分享几个最常见的错误和解决方法。
```

**[0:20-1:00] 错误 1：command not found**
```
如果提示找不到命令，
说明没有激活虚拟环境。

解决方法：
source .venv/bin/activate

看到提示符前面有 (.venv) 就对了。
```

**[1:00-1:40] 错误 2：找不到文件**
```
文件路径错误很常见。

记住两点：
1. 文件要放在 pass 文件夹
2. 文件名要完全匹配，包括中文和空格

检查文件：
ls pass/

使用 Tab 键自动补全文件名。
```

**[1:40-2:20] 错误 3：依赖安装失败**
```
如果安装依赖时报错：

1. 检查网络连接
2. 升级 pip：pip3 install --upgrade pip
3. 重新安装：pip3 install -r requirements.txt

如果还不行，删除虚拟环境重新创建：
rm -rf .venv
python3 -m venv .venv
```

**[2:20-3:00] 总结和求助**
```
大部分问题都能通过这些方法解决。

如果还是不行：
1. 查看详细文档
2. 检查 GitHub Issues
3. 提交新 Issue，附上错误信息

记住，遇到问题是正常的，
不要放弃，一步步排查就能解决！
```

## 📝 录制建议

### 录制环境
- **分辨率**：1920x1080 或 1280x720
- **字体大小**：终端字体调大（16-18pt）
- **录屏工具**：
  - macOS: QuickTime / ScreenFlow
  - Windows: OBS Studio / Camtasia
  - Linux: SimpleScreenRecorder

### 录制技巧
1. **说话速度**：放慢，给观众思考时间
2. **暂停时机**：重要操作前后暂停 1-2 秒
3. **鼠标提示**：用红圈或箭头标注重点
4. **字幕**：添加中文字幕（可选）
5. **章节标记**：在视频中添加章节

### 后期处理
- 剪掉等待时间（安装依赖、网络请求）
- 添加背景音乐（轻柔，音量小）
- 添加片头片尾
- 标注关键命令（文字叠加）
